# i18n Setup Guide

Complete guide for implementing internationalization in PATA & CÃO.

## Quick Start

### 1. Install Dependencies

```bash
cd frontend
npm install i18next react-i18next i18next-browser-languagedetector
```

### 2. Create Directory Structure

```bash
mkdir -p src/locales/{es,pt,pt-BR,en}
touch src/locales/{es,pt,pt-BR,en}/translation.json
touch src/i18n.config.ts
```

### 3. Initialize i18n Configuration

Create `src/i18n.config.ts`:

```typescript
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import LanguageDetector from 'i18next-browser-languagedetector';

// Import translation files
import esTranslation from './locales/es/translation.json';
import ptTranslation from './locales/pt/translation.json';
import ptBRTranslation from './locales/pt-BR/translation.json';
import enTranslation from './locales/en/translation.json';

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    fallbackLng: 'pt-BR',
    ns: ['translation'],
    defaultNS: 'translation',
    resources: {
      es: { translation: esTranslation },
      pt: { translation: ptTranslation },
      'pt-BR': { translation: ptBRTranslation },
      en: { translation: enTranslation },
    },
    detection: {
      order: ['localStorage', 'navigator', 'htmlTag'],
      caches: ['localStorage'],
    },
    interpolation: {
      escapeValue: false, // React already escapes values
    },
  });

export default i18n;
```

### 4. Initialize in App Entry Point

In `src/main.tsx` or your app's entry:

```typescript
import './i18n.config'; // Import before rendering
import App from './App';

ReactDOM.createRoot(document.getElementById('root')!).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);
```

### 6. Create Language Selector Component

**`src/components/LanguageSelector.tsx`**:

```typescript
import { useTranslation } from 'react-i18next';

export function LanguageSelector() {
  const { i18n, t } = useTranslation();

  const languages = [
    { code: 'pt-BR', label: 'Português (Brasil)', flag: '🇧🇷' },
    { code: 'pt', label: 'Português', flag: '🇵🇹' },
    { code: 'es', label: 'Español', flag: '🇪🇸' },
    { code: 'en', label: 'English', flag: '🇬🇧' },
  ];

  return (
    <select
      value={i18n.language}
      onChange={(e) => i18n.changeLanguage(e.target.value)}
      aria-label={t('common.language')}
    >
      {languages.map((lang) => (
        <option key={lang.code} value={lang.code}>
          {lang.flag} {lang.label}
        </option>
      ))}
    </select>
  );
}
```

### 7. Use Translations in Components

**Example Component**:

```typescript
import { useTranslation } from 'react-i18next';

export function PetProfile({ petId }: { petId: string }) {
  const { t, i18n } = useTranslation();
  const [pet, setPet] = useState(null);

  useEffect(() => {
    fetchPet(petId).then(setPet);
  }, [petId]);

  if (!pet) return <div>{t('common.loading')}</div>;

  return (
    <div>
      <h1>{t('pet.petProfile')}</h1>
      <p>{t('pet.petName')}: {pet.name}</p>
      <p>{t('pet.breed')}: {pet.breed}</p>
      <p>{t('pet.age')}: {pet.age}</p>
      
      <h2>{t('pet.healthRecords')}</h2>
      <ul>
        {pet.healthRecords?.map((record) => (
          <li key={record.id}>{record.description}</li>
        ))}
      </ul>

      <p>{t('common.currentLanguage')}: {i18n.language}</p>
    </div>
  );
}
```

## Advanced Features

### Pluralization

Translate plural forms based on count:

```json
{
  "pet": {
    "petCount_one": "Você tem {{count}} pet",
    "petCount_other": "Você tem {{count}} pets"
  }
}
```

Use in component:

```typescript
const { t } = useTranslation();
t('pet.petCount', { count: 5 }); // "Você tem 5 pets"
t('pet.petCount', { count: 1 }); // "Você tem 1 pet"
```

### Interpolation

Include variables in translations:

```json
{
  "booking": {
    "bookingWith": "Agendamento com {{providerName}} em {{date}}"
  }
}
```

```typescript
t('booking.bookingWith', {
  providerName: 'João',
  date: new Date().toLocaleDateString('pt-BR'),
});
```

### Date & Time Formatting

Use `Intl` API for locale-aware formatting:

```typescript
const dateFormatter = new Intl.DateTimeFormat(i18n.language, {
  year: 'numeric',
  month: 'long',
  day: 'numeric',
});

dateFormatter.format(new Date()); // "15 de maio de 2026" (pt-BR)
```

### Currency Formatting

```typescript
const currencyFormatter = new Intl.NumberFormat(i18n.language, {
  style: 'currency',
  currency: 'BRL', // or EUR, USD
});

currencyFormatter.format(150.50); // "R$ 150,50" (pt-BR)
```

## Testing i18n

### Unit Test: Key Completeness

```typescript
// src/locales/__tests__/i18n.test.ts
import es from '../es/translation.json';
import pt from '../pt/translation.json';
import ptBR from '../pt-BR/translation.json';

describe('i18n translations', () => {
  it('should have identical keys across all languages', () => {
    const esKeys = JSON.stringify(Object.keys(es).sort());
    const ptKeys = JSON.stringify(Object.keys(pt).sort());
    const ptBRKeys = JSON.stringify(Object.keys(ptBR).sort());

    expect(esKeys).toBe(ptKeys);
    expect(esKeys).toBe(ptBRKeys);
  });

  it('should not have empty translations', () => {
    const allTranslations = [es, pt, ptBR];
    allTranslations.forEach((trans) => {
      Object.entries(trans).forEach(([key, value]) => {
        if (typeof value === 'string') {
          expect(value).not.toBe('');
        }
      });
    });
  });
});
```

### E2E Test: Language Switching

```typescript
// e2e/language-switcher.cy.ts
describe('Language Switcher', () => {
  beforeEach(() => {
    cy.visit('/');
  });

  it('should switch to Spanish', () => {
    cy.get('[aria-label*="Idioma"]').select('es');
    cy.contains('Buscar Prestadores'); // Spanish text
    cy.contains('Adicionar Pet').should('not.exist'); // Portuguese gone
  });

  it('should persist language preference', () => {
    cy.get('[aria-label*="Idioma"]').select('es');
    cy.reload();
    cy.get('[aria-label*="Idioma"]').should('have.value', 'es');
  });

  it('should use Brazilian Portuguese by default', () => {
    cy.contains('Adicionar Pet'); // Brazilian Portuguese default
  });
});
```

## Deployment Checklist

Before releasing a new version:

- [ ] All UI text is in translation files (no hardcoded strings in JSX)
- [ ] All 4 language files have identical key structures
- [ ] No empty translation values
- [ ] Date/time/currency formatted with locale awareness
- [ ] Tests pass for all languages
- [ ] E2E tests cover language switching
- [ ] RTL languages handled (if needed in future)
